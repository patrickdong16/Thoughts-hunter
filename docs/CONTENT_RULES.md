# 思想雷达内容规范 (Content Rules v4.0)

本文件定义内容生成与发布的核心规范。**每次生成内容前必须参照此文件。**

---

## � 发布日类型 (Day Types)

思想雷达将每日发布分为两种类型：**普通日** 和 **主题日**。

| 类型 | 标识 | 说明 |
|------|------|------|
| **普通日 (Regular Day)** | 🔵 | 常规运营模式，均衡覆盖六大频段 |
| **主题日 (Theme Day)** | 🔴 | 针对重大国际活动的专题聚焦模式 |

---

## 🔵 普通日规则 (Regular Day Rules)

适用于：日常运营，无重大事件时期

### 内容配额

| 项目 | 要求 |
|------|------|
| **总数量** | 6-8 条/天 |
| **频段分布** | T/P/H/Φ/F/R 各1条 (1:1:1:1:1:1) |
| **视频内容** | 1-2 条/天 |
| **字符要求** | 生成≥700字 / 验证≥300字 |

### 来源多样性
- 必须覆盖不同作者、不同媒体
- 避免单一来源主导

---

## 🔴 主题日规则 (Theme Day Rules)

适用于：重大国际论坛、峰会、大会、历史性事件期间

### 触发条件

以下情况可启用主题日模式：

| 类别 | 示例事件 |
|------|---------|
| **国际论坛/峰会** | 达沃斯世界经济论坛、APEC峰会、G20峰会、联合国大会 |
| **科技大会** | CES、MWC、Google I/O、Apple WWDC、OpenAI DevDay |
| **学术/政策会议** | 诺贝尔周、慕尼黑安全会议、博鳌亚洲论坛 |
| **重大历史事件** | 领导人会晤、重大政策发布、历史性协议签署 |
| **行业里程碑** | AGI突破性进展、重大产品发布、市场剧变 |

### 内容配额（放宽限制）

| 项目 | 主题日规则 | 与普通日对比 |
|------|-----------|-------------|
| **总数量** | 20-30 条/天 | ↑ 主题日专属高配额 |
| **最低要求** | 20 条/天 | ⚠️ 必须达到下限 |
| **频段分布** | **不限** | 🔓 解除 1:1:1:1:1:1 限制 |
| **视频内容** | **不限** | 🔓 可全部为视频 |
| **视频时长** | ≥20 分钟 | ↓ 放宽（普通日≥40分钟）|
| **来源类型** | 聚焦主题 | 允许密集同源 |

### 🌐 拓展来源规则 (Expanded Sources)

> 🔑 **核心原则**: 主题日**不限于预设人物名单**，任何在活动中发表高质量/有影响力观点的领袖都可入选。

| 类别 | 可入选人物示例 |
|------|--------------|
| **国家领导人** | 各国元首、政府首脑、部长级官员 |
| **央行/财金** | 央行行长、财政部长、IMF/世行负责人 |
| **国际组织** | 联合国官员、WTO总干事、WHO负责人 |
| **商业领袖** | 跨国公司CEO、行业巨头创始人 |
| **学术/智库** | 顶尖学者、智库负责人、诺奖得主 |
| **科技大佬** | AI实验室负责人、科技公司CEO |
| **媒体/艺术** | 知名主编、文化评论家、艺术领袖 |

⚠️ **配置文件中的 `keyFigures` 是参考列表，而非限制名单**。实际内容选取时，应：
- 优先关注配置中的重点人物
- 同时捕捉任何发表重要观点的相关领域领袖
- 以内容质量和影响力为最终标准

### 🚨 主题日核心法则 (Critical Theme Day Mandates)

> [!CAUTION]
> **以下两条法则为主题日最高准则，必须严格遵守**

#### 法则一：100% 主题聚焦 (Pure Theme Focus)

主题日期间发布的**每一条内容**都必须直接来自该主题活动：

| 要求 | 说明 |
|------|------|
| **活动相关性** | 必须来自主题活动现场演讲、访谈、圆桌、新闻发布会 |
| **热度优先** | 以 YouTube 视频的观看量和互动数为优先级，**绝不能错过高热度内容** |
| **质量保证** | 优先选择深度访谈、主题演讲等高信息密度内容 |
| **零偏离容忍** | 非主题活动内容（即使是预设人物的其他言论）一律不收录 |

**执行标准**：
```
✅ 正确：Mark Carney 在达沃斯论坛的演讲
✅ 正确：Jensen Huang 在达沃斯的采访
❌ 错误：Elon Musk 在 X 上发布的推文（非现场活动）
❌ 错误：Sam Altman 上周的播客访谈（非主题日活动）
```

#### 法则二：跨领域领袖资格 (Multi-Domain Leadership)

主题日内容发布人**必须是各领域公认领袖**：

| 领域 | 资格标准 |
|------|---------|
| **政治** | 国家元首、政府首脑、部长、大使 |
| **科技** | AI/Tech 公司创始人/CEO、顶级研究员 |
| **金融** | 央行行长、大型银行/基金CEO、财政部长 |
| **经济** | 顶级经济学家、国际组织负责人 |
| **学术** | 顶尖大学校长/教授、诺奖得主 |
| **媒体** | 权威媒体主编、知名评论员 |
| **艺术** | 文化机构负责人、国际知名艺术家 |

> ⚠️ **普通评论员、一般记者、非知名分析师的言论不符合主题日收录标准**

```json
  }

```

## 🚨 内容去重规范 (Duplicate Prevention)

> ⚠️ **此规则对普通日和主题日均强制执行**

### 核心原则

- **零重复容忍**：已发布的内容**严禁再次发布**
- **来源唯一性**：同一原始来源（URL/YouTube ID）**仅允许发布一次**
- **唯一性校验前置**：所有内容写入前必须通过校验

### 去重检查清单

| 检查项 | 阻断级别 |
|-------|---------|
| 来源URL/视频ID 已存在 | **P0 强制阻断** |
| 标题相似度 ≥ 80% | **P0 强制阻断** |
| 同作者同频段7天内相似观点 | **P1 需审核** |
| 同日期同主题（普通日） | **P2 警告** |

## 🔍 来源质量标准（通用）

### ✅ 合格来源
- 权威榜单/奖项获得者（诺贝尔、普利策、图灵奖等）
- 权威媒体深度栏目（NYT、FT、The Atlantic、Foreign Affairs）
- 顶级头脑原始表达（TED、达沃斯、Lex Fridman播客等）
- 【主题日额外】活动官方频道、指定媒体合作伙伴

### ❌ 禁止使用
- 普通新闻报道（无深度分析）
- 社交媒体碎片化言论
- 未经验证的自媒体内容
- 商业公关/软文

---

## ⚠️ 真实性要求（不可妥协）

- **零容忍虚构**：严禁虚构来源、日期、引用或事实
- **可追溯性**：每条内容必须关联真实原始链接
- **双重核验**：自动化草稿需手动确认来源存在

## 📍 来源信息格式规范 (Source Information v5.0)

> 🆕 **v5.0 新增**：每条内容必须包含完整的来源信息，支持用户验证真实性

### 来源字段定义

| 字段 | 必填 | 说明 | 示例 |
|------|------|------|------|
| `source` | ✅ 是 | 发布场合、活动名称和时间 | `Davos WEF 2026 主题演讲, 2026年1月23日` |
| `source_url` | ✅ 是 | 可访问的原文/视频链接 | `https://youtube.com/watch?v=xxx` |

### source 字段格式

**格式模板**：`[活动/平台名称] [类型], [日期]`

| 类型 | 格式示例 |
|------|----------|
| 论坛演讲 | `Davos WEF 2026 主题演讲, 2026年1月23日` |
| 访谈节目 | `Lex Fridman Podcast #420, 2026年1月15日` |
| 新闻采访 | `Bloomberg TV 专访, 2026年1月20日` |
| 学术讲座 | `MIT Technology Review 年会, 2026年1月10日` |
| 播客节目 | `Acquired Podcast Episode 180, 2026年1月12日` |

### source_url 要求

- **必须可访问**：链接必须能够打开
- **必须是原始来源**：不接受二手转载链接
- **优先选择官方渠道**：YouTube 官方频道 > 第三方上传

**合格 URL 示例**：
```
✅ https://www.youtube.com/watch?v=AbCdEfGhIjK
✅ https://www.foreignaffairs.com/articles/xxx
✅ https://www.bloomberg.com/news/videos/xxx
❌ https://twitter.com/xxx (碎片化内容)
❌ https://unknown-blog.com/xxx (非权威来源)
```

### 内容录入检查清单

发布前必须确认：

- [ ] `source` 字段已填写（含活动名称 + 日期）
- [ ] `source_url` 字段已填写（可点击访问）
- [ ] URL 已验证可访问
- [ ] URL 指向原始来源（非转载）
