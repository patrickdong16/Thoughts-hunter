# 思想雷达内容质量控制规范

> ⚠️ **强制执行**：所有内容生成必须遵循本规范，违反规则的内容不得发布。

## 一、内容真实性要求

### 1.1 禁止虚构（零容忍）

- ❌ **禁止虚构来源**：不得编造不存在的文章、演讲、论文
- ❌ **禁止虚构引用**：不得编造作者未说过的话
- ❌ **禁止虚构发表时间**：来源日期必须准确
- ❌ **禁止事实性错误**：如"乌克兰战争两周年"（实际已4年）

### 1.2 必须可验证

每条内容必须满足以下至少一项：
1. **URL可访问**：提供原始链接，内容真实存在
2. **视频可查证**：YouTube视频ID真实有效
3. **出版物可检索**：期刊/书籍可在公开数据库查到

## 二、来源标注规范

### 2.1 格式要求

```
[发表日期] · [出处名称] · [具体场合/栏目]
```

### 2.2 正确示例

```
✅ 2024年3月15日 · Lex Fridman Podcast #789 · 与Andrej Karpathy对话
✅ 2025年11月 · Foreign Affairs · 专栏文章"The Return of Great Power Competition"
✅ 2025年1月10日 · TED Talk · 年度大会主题演讲
```

### 2.3 错误示例

```
❌ 2026年1月19日 · Foreign Affairs · 基于乌克兰战争两周年的回顾分析
   问题：虚构文章，且事实错误（战争已4年）
   
❌ 2026年1月20日 · YouTube深度讲座 · 关于AI发展路径的个人见解
   问题：无具体视频链接，无法验证
```

## 三、自动化内容生成质量控制

### 3.1 生成前检查

1. **来源验证**：Claude 必须基于真实存在的视频/文章分析
2. **时间校验**：自动验证事件时间线是否合理
3. **人物验证**：确认作者身份和所属机构正确

### 3.2 生成后验证

在 `/api/automation/generate-daily` 流程中增加：

```javascript
// 验证函数示例
function validateContent(item) {
  const errors = [];
  
  // 1. 检查来源格式
  if (!item.source || item.source.length < 20) {
    errors.push('来源描述过短');
  }
  
  // 2. 检查是否有可验证的URL/视频ID
  if (!item.video_url && !item.article_url) {
    errors.push('缺少可验证的原始链接');
  }
  
  // 3. 时间合理性检查
  const sourceDate = extractDate(item.source);
  if (sourceDate && sourceDate > new Date()) {
    errors.push('来源日期在未来');
  }
  
  return errors;
}
```

### 3.3 发布控制

- **自动发布条件**：必须基于真实YouTube视频字幕分析
- **人工审核条件**：任何非YouTube来源的内容需人工确认

## 四、手动添加内容规范

### 4.1 必填字段验证

| 字段 | 验证规则 |
|------|----------|
| source | 必须包含可验证的日期和出处 |
| content | ≥500字，结构完整 |
| video_url / article_url | 至少一个有效链接 |
| author_bio | 真实可查的作者身份 |

### 4.2 禁止行为

1. **禁止"编造式创作"**：不得为了填充内容而虚构文章
2. **禁止"张冠李戴"**：不得将A的观点归于B
3. **禁止"时间穿越"**：不得引用尚未发生的事件

## 五、错误内容处理

### 5.1 发现错误后

1. **立即下架**：删除错误内容
2. **分析原因**：记录错误类型和根本原因
3. **更新规则**：将新错误类型加入检查清单

### 5.2 错误日志

维护 `/docs/content-errors.log`，记录：
- 错误内容ID
- 错误类型
- 发现时间
- 修复措施
