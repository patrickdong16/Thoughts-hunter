# 思想雷达项目规范 (Thoughts Radar Rules)

本文件定义了思想雷达项目的核心规范，所有内容创建、UI设计、开发决策必须遵循这些规则。

---

## 一、内容来源筛选规范

### 1.1 三类合格来源（可并存）

#### 类型1：权威与共识的筛选机制
- **权威榜单/奖项**：诺贝尔奖、普利策奖、图灵奖、布克奖等国际顶级奖项的获得者/作品
- **权威媒体深度栏目**：New York Times、Financial Times、The Atlantic、Foreign Affairs、The Economist 的长篇深度文章
- **高质量出版社与机构推荐**：哈佛大学出版社、普林斯顿大学出版社、牛津大学出版社等学术出版机构

**特征**：非虚构为主，强调严肃性、人文性与论证结构

#### 类型2：顶级头脑的原始表达（当代与历史）
- **书籍**：作者本人作品优先，如尤瓦尔·赫拉利《人类简史》、韩炳哲《倦怠社会》
- **长访谈/演讲/播客/YouTube**：TED演讲、达沃斯论坛演讲、Lex Fridman播客、Making Sense Podcast等
- **深度文章与长评论**：NYT Opinion、FT专栏、Foreign Affairs 特稿、The New Yorker 长文

**要求**：保持原语言 + 原链接，语言不限

#### 类型3：对现实有解释力、能进入命题对照的思想材料
- **不是**：新闻事实报道
- **是**：对事实背后的结构解释、理论框架与价值冲突的表达

**核心原则**：思想雷达追踪的是**原始思想表达**，中文只是"解读层"和"对照层"。

---

### 1.2 不合格来源示例（禁止使用）

- ❌ 普通新闻报道（无深度分析）
- ❌ 社交媒体碎片化言论
- ❌ 未经验证的自媒体内容
- ❌ 转述/二手解读（除非标注原始来源）
- ❌ 商业公关/软文
- ❌ 缺乏论证结构的观点陈述

## 二、内容质量规范

### 2.1 内容结构（必须完整）

每条雷达内容必须包含以下字段：

| 字段 | 要求 | 示例 |
|------|------|------|
| **freq** | 频段ID | T1, P2, Φ3, F1... |
| **stance** | 立场 | A 或 B |
| **title** | 总结性标题（观点鲜明） | "AI精英化正在加速：顶级人才争夺战的社会后果" |
| **author_name** | 作者全名 | Daron Acemoglu |
| **author_avatar** | 头像缩写（2字符） | DA |
| **author_bio** | 作者简介 + 主要作品/思想 | "MIT经济学教授，2024年诺贝尔经济学奖得主" |
| **source** | 发表时间 + 出处/场合 | "2025年1月 · 《美国经济评论》特稿 · 获诺奖后首篇系统论述" |
| **content** | 正文（见2.2） | 详见下方 |
| **tension_q** | 张力问题 | "AI是否正在重写社会分层结构？" |
| **tension_a** | A极立场描述 | "AI普惠化，整体能力上移" |
| **tension_b** | B极立场描述 | "AI精英化，阶层固化" |
| **keywords** | 关键词数组（3-5个） | ["AI", "社会分层", "认知寡头制"] |

### 2.2 正文质量要求

#### 字数要求
- **最低**：500字
- **推荐**：800-1500字
- **充分论证的内容**：1000字以上

#### 结构要求（必须包含）

1. **核心观点**（第一段）
   - 明确陈述作者的主要判断/论点
   - 使用第一人称自述风格或权威第三人称转述

2. **主要论据**（中间段落）
   - 2-4个支撑论点
   - 每个论点需要具体案例或数据支撑
   - 逻辑链条清晰

3. **社会影响/结论**（最后段落）
   - 政策建议、预测或呼吁
   - 对读者的启示

#### 写作风格要求

- ✅ 自述风格：作者陈述自己的观点（如"Acemoglu在这篇论文中提出..."）
- ✅ 保留原始论证结构
- ✅ 使用段落分隔（\\n\\n）
- ✅ 专业术语需解释

- ❌ 简单罗列观点
- ❌ 缺乏论证的结论
- ❌ 情绪化表达
- ❌ 模糊笼统的描述

## 四、UI设计规范

### 4.1 主题颜色

```css
/* 主色 */
--primary: #00ff88;        /* 霓虹绿 */

/* 背景 */
--background: #08090c;     /* 纯黑 */

/* 文本 */
--text: #ffffff;           /* 主文本 */
--text-secondary: #9ca3af; /* 次要文本 */
--text-muted: #6b7280;     /* 弱化文本 */

/* 立场颜色 */
--side-a: #4a9eff;         /* A极 - 蓝色 */
--side-b: #f0a500;         /* B极 - 琥珀色 */

/* 边框 */
--border: #1f2937;
--card-border: rgba(0, 255, 136, 0.1);

/* 领域颜色 */
--domain-tech: #4a9eff;      /* 科技 */

--domain-philosophy: #ec4899;/* 哲学 */
--domain-religion: #f59e0b;  /* 宗教 */
--domain-finance: #10b981;   /* 金融 */
```

### 4.2 卡片结构

每张雷达卡片必须按以下顺序显示：

1. **标签区**
   - 频段标签（如T1, P2）- 使用领域颜色
   - 立场标签（倾向A/B）- 使用蓝/琥珀色

2. **标题**
   - 字号20px，加粗
   - 颜色：白色

3. **作者信息**
   - 圆形头像（绿色边框）+ 2字符缩写
   - 作者名 + 简介

4. **来源**
   - 绿色左边框标识

5. **正文**
   - 展开/收起功能
   - 行高1.8

6. **张力信息区**
   - 深色背景卡片
   - 问题居中
   - A极（蓝）| B极（琥珀）对比显示

7. **关键词标签**
   - 绿色边框小标签

8. **操作区**
   - 收藏按钮（❤️）
   - 我倾向A / 我倾向B 按钮

### 4.3 字体规范

- **标题**：24px, 700
- **卡片标题**：20px, 700
- **正文**：15px, 400, line-height: 1.8
- **次要文本**：13px, 500
- **标签**：11px, 700
- **等宽字体**：JetBrains Mono（用于频段ID、日期等）

## 六、高质量内容示例

### 示例1：符合规范

```
频段: T1
立场: B
标题: AI精英化正在加速：顶级人才争夺战的社会后果
作者: Daron Acemoglu
简介: MIT经济学教授，2024年诺贝尔经济学奖得主
出处: 2025年1月 · 《美国经济评论》特稿 · 获诺奖后首篇系统论述AI社会影响的学术论文

正文:
Acemoglu在这篇万字长文中提出了一个令人不安的判断：AI技术正在创造他所谓的「认知寡头制」。这不是一个修辞，而是基于对过去十年科技行业人才流动数据的系统分析得出的结论。全球顶级AI人才高度集中于不到20家公司，技术能力差距正以指数级速度拉开。

论证建立在三个相互强化的机制上。第一是「技能溢价的非线性跃升」：AI带来的不是线性的技能溢价，而是断层式的能力鸿沟。第二是「教育系统的结构性滞后」：即使今天立即启动教育改革，培养一代新人也需要15到20年。第三是「精英再生产的闭环化」：掌握AI的家庭可以为下一代提供无法复制的教育资源。

Acemoglu直接反对「让市场自行调节」的乐观主义。他预测，如果不加干预，社会将在10到15年内形成事实上的「双轨制」——少数AI精英与大多数「被管理者」。他提出的政策建议包括：对AI公司征收「认知租金税」，用于资助全民基本收入和终身教育项目。

关键词: AI, 社会分层, 认知寡头制, 诺贝尔经济学奖
```
